# Концепция: страница = блок

> Документ-конституция проекта. Все шаблоны, инструменты и правки сверяются с ним.
> Обновлено: 2026-06-11.
>
> Механизм исполнения — `docs/SPEC_PAGE_FILL.md`: паспорта страниц шаблонов
> (page_contract в styles.json), бюджет ёмкости (estimate_page_budget),
> постраничный QC каждой компиляции (layout_report, strict_layout).

## 1. Главный принцип

**Единица дизайна — страница A4 (или A5 для книги), а не абзац и не картинка.**

Страница — это холст с жёсткими краями, как слайд. Движок обязан:

1. **Думать заранее.** До рендера посчитать, сколько текста и картинок ляжет на ОДНУ
   страницу (measure по реальной ширине колонки и высоте полосы).
2. **Заполнить полосу до низа.** Большая пустота внизу допустима только если контент
   действительно закончился (конец главы в книге — норма; дыра в середине журнальной
   статьи — брак).
3. **Перенести остаток чисто.** Всё, что не влезло, уходит на следующую страницу как
   обычный текст: без нового заголовка, без повторных картинок, можно с разрывом
   предложения и даже на переносе — как в книге.
4. **Только потом проектировать следующую страницу.** Идеал первой → переход ко второй.

Антипаттерн, который этот принцип запрещает: накидать независимые блоки
(заголовок-блок, картинка-блок, текст-блок) друг на друга и «как распределилось,
так распределилось». Так делает любая нейросеть без MCP; ценность MCP — именно
в дисциплине страницы.

## 2. Иллюстрации: два разных случая — два разных инструмента

| Случай | Признак | Раскладка |
|---|---|---|
| **Одна история, несколько фото** | один заголовок | Текст — ОДИН непрерывный поток; фото «влиты как кирпичики» по диагонали (первое вверху на одной стороне, второе ниже на противоположной). Никакой сегментации текста по фото — иначе рваный край и дыры. |
| **Несколько мини-историй** | у каждого фото свой заголовок | Каждая история — свой блок «заголовок + фото + текст», и движок компонует 2 истории на одной странице как целостную полосу. |

Прочие правила иллюстраций:

- Раскладка выбирается от **пропорций фото** (aspect = height/width): широкая
  горизонталь → во всю ширину по центру; портрет/квадрат → обтекание сбоку.
- **Журнал:** подписи по умолчанию НЕТ; если задана — по центру под фото.
- **Книга:** обтекания сбоку НЕ существует. Картинка строго по центру колонки,
  подпись опциональна, текст продолжается ниже. (Сбоку — только буквица.)
- **ГОСТ:** рисунки по центру, подпись «Рисунок N — …» снизу по центру.

## 3. Типографика: ровный край без рек

- **Justify оставляем** везде, где он нужен (журнал, книга, отчёт, портфолио, ГОСТ):
  ровный правый край — признак профессиональной вёрстки.
- **Реки** (огромные зазоры между словами) лечатся не отказом от justify, а плотными
  переносами: `set text(costs: (hyphenation: 5%, ...))` — строки набиваются плотно,
  растягивать нечего. Проверено: чистая полоса и колонка у фото набираются ровно.
- Реки — общая болезнь justify (бывает и без картинок); обтекание лишь обостряет её.
- `wrap-top-bottom` из wrap-it **запрещён**: он принудительно растягивает строку на
  стыке → гарантированная река. Диагональ собирается из двух обычных `wrap-content`
  (или meander, см. §6).
- **technical / letter** — по левому краю осознанно (их стандарт).
- **academic_ru (ГОСТ)** — justify по ГОСТу, переносы выключены (нормоконтроль их
  часто запрещает). Не «делать красиво», а делать ПО СТАНДАРТУ.

## 3a. Авто-масштаб: страница = блок в ОБЕ стороны

Если контента не хватает на полный лист, одностраничный шаблон не имеет права
оставлять «сжатое уродство сверху и дыру снизу». Шаблон строится как функция
масштаба s: перед рендером он МЕРЯЕТ себя на наборе масштабов и выбирает самый
крупный, при котором всё ещё умещается в один лист — шрифты, воздух, элементы
растут пропорционально, лист заполняется красиво.

- **resume**: кандидаты 1.3 → 1.0 (резюме может смело укрупняться);
- **letter**: кандидаты 1.15 → 1.0 (деловое письмо обязано остаться сдержанным);
- если даже 1.0 не влезает — честный переход на страницу 2, без ужимания.

Реализация: `#let cv(s) = {...все размеры × s, вертикали в em...}` +
`#context { for s in candidates { if measure(box(width: avail-w, cv(s))).height
<= avail-h { pick = s; break } } cv(pick) }`.

## 4. Редактирование без «разъезжания»

PDF никогда не правится как PDF. **Источник истины — JSON-состояние документа**
(секции, картинки, пресет), а PDF — детерминированная проекция этого состояния.

- Правка = `update_section()` / `add_image()` → пере-рендер → движок ЗАНОВО
  пересчитывает страницы по правилам §1–§3. Ничего не может «разъехаться»,
  потому что раскладка не хранится — она каждый раз выводится из правил.
- `save_document()` / `load_document()` делают это переживаемым между сессиями.
- Следствие: «подправить подпись» или «заменить фото» — операция над JSON,
  а движок сам решит, как заново уложить страницы.

## 5. Контроль качества (обязательный цикл)

1. `compile_preview()` → PNG каждой страницы.
2. Чеклист: заполнена ли полоса до низа; нет ли рек; нет ли висячих строк/заголовка
   у нижнего края; фото на своих местах; подписи по правилам шаблона.
3. Машинная проверка пустоты: PIL+numpy, последняя «строка с чернилами» на странице
   (см. tests/render_all.py). Дыра > ~150px в середине материала = брак.
4. Только после визуальной проверки — `compile_pdf()`.

## 6. Ресёрч 2026-06: что берём на вооружение

### meander (@preview/meander:0.4.3, обновлён 2026-05-28) — кандидат №1 для журнала

Полноценный движок «страница = блок»: страница режется на зоны, `placed()` ставит
препятствия (фото) в любые точки полосы, `container()` задаёт зону текста,
`content[]` — единый поток, который сам протекает между препятствиями.

Проверено локально (Typst 0.14.2, /tmp/mtest):
- Диагональ «фото вверху-справа + фото внизу-слева, один текст вокруг обоих» —
  собирается ОДНИМ вызовом, текст обтекает нижнее фото и сбоку (wrap-it так не умеет);
- полоса заливается до низа, justify+переносы работают;
- многостраничность: `opt.overflow.repeat()` продолжает текст на следующих страницах
  (но клонирует и фото — для нас не годится); правильный путь —
  `opt.overflow.custom(...)`: забрать остаток и вылить обычным потоком без картинок.

План: переписать журнальный разворот (а позже сложные полосы report/portfolio)
с wrap-it на meander. wrap-it остаётся для простого одиночного обтекания.

### ГОСТ: реферат «под ключ» — достижимо стандартными средствами

- **Библиография:** в Typst встроен стиль `gost-r-705-2008-numeric` —
  `bibliography("sources.yml", style: "gost-r-705-2008-numeric")`, источники в
  Hayagriva YAML. Список уходит на отдельную страницу «Список использованных
  источников». Нужен новый MCP-инструмент: `add_bibliography(doc_id, sources)`.
- **Референс оформления:** пакет `modern-g7-32` (ГОСТ 7.32-2017) — титульный лист,
  реферат, содержание, нумерация рисунков/таблиц/приложений, 1.5 интервала.
  Брать как образец требований для развития academic_ru (не как замену шаблона).
